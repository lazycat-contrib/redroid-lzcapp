GPU 加速 Android 云方案

ReDroid

Remote anDroid — 在容器中启动 Android 实例，支持 GPU 硬件加速。 适用于云游戏、虚拟手机、自动化测试等场景，兼容 arm64 和 amd64 架构。

安全警告 切勿将 ADB 端口暴露在公网上。任何获得 ADB 访问权限的人都可以完全控制该 Android 实例，甚至可能危及宿主机安全。请仅在可信的本地网络或 VPN 内连接。

核心特性

GPU 硬件加速

通过宿主机 GPU 进行硬件加速渲染，流畅运行图形应用和游戏。

多实例支持

单台 Linux 主机上同时启动多个相互隔离的 Android 实例。

arm64 & amd64

原生支持两种 CPU 架构，兼容 Docker、Podman、Kubernetes 等容器环境。

Android 8.1 – 16

覆盖从经典版本到最新 Android 的广泛版本支持。

通过 ADB 连接

  1. 在本地安装 ADB

    从 Android Developer Studio 下载，或使用包管理器安装。

    Shell
    # Debian / Ubuntu
sudo apt install adb

# macOS
brew install android-platform-tools

# Windows — 从以下地址下载 SDK Platform Tools：
# https://developer.android.com/studio#downloads

  2. 获取连接地址

    连接地址为当前页面的域名 + ADB 端口 5555（已在页面顶部显示）。

  3. 执行连接命令

    Shell
    adb connect HOST:5555

# 验证连接状态
adb devices
# 正常输出：
# HOST:5555    device
如果连接失败，请确认 ADB 端口在网络上可达。检查防火墙规则，确保你和设备处于同一局域网或 VPN 中。

使用 scrcpy 投屏控制

  1. 安装 scrcpy

    Shell
    # Debian / Ubuntu
sudo apt install scrcpy

# macOS
brew install scrcpy

# Windows
# 从 https://github.com/Genymobile/scrcpy/releases 下载安装

  2. 启动 scrcpy

    Shell
    scrcpy -s HOST:5555

# 常用选项：
scrcpy -s HOST:5555 --max-size 1024     # 限制分辨率
scrcpy -s HOST:5555 --bit-rate 2M        # 限制码率
scrcpy -s HOST:5555 --no-audio            # 不转发音频
scrcpy -s HOST:5555 --stay-awake          # 保持屏幕常亮
scrcpy -s HOST:5555 --video-codec h265    # H.265 硬件编码
网络较慢时建议使用 --max-size 1024 --bit-rate 2M 降低带宽占用。scrcpy 默认使用 H.264 编码，使用 --video-codec h265 可在更低码率下获得更好画质。

ADB 命令速查

命令说明
adb devices列出已连接设备
adb shell打开设备交互式 Shell
adb install app.apk安装 APK 文件
adb uninstall <包名>按包名卸载应用
adb push local /sdcard/推送文件到设备
adb pull /sdcard/f local从设备拉取文件
adb reboot重启 Android 实例
adb logcat查看实时系统日志
adb shell wm size获取屏幕分辨率
adb shell wm density获取屏幕 DPI
adb disconnect <地址>断开设备连接
adb kill-server重启 ADB 服务

常见问题

ADB 连接被拒绝

  • 确认 redroid 容器正在运行中
  • 检查连接地址和端口是否正确
  • 确认防火墙没有拦截 TCP ingress 端口
  • 尝试 adb kill-server 后重新连接

scrcpy 显示黑屏

  • 等待 1–2 分钟让 Android 系统完全启动
  • 尝试 adb reconnect 刷新连接
  • 确认容器分配了足够的内存

容器无法启动

  • 宿主机必须加载 binder_linux 和 ashmem_linux 内核模块
  • 验证命令：lsmod | grep -E "binder|ashmem"
  • 宿主机必须支持特权容器
  • 查看容器日志获取详细错误信息

宿主机要求

内核模块说明 LazyCat 宿主机需要加载必需的内核模块。如果容器启动失败，管理员可能需要在宿主机上执行以下命令：
Shell（宿主机）
apt install linux-modules-extra-$(uname -r)
modprobe binder_linux devices="binder,hwbinder,vndbinder"
modprobe ashmem_linux
