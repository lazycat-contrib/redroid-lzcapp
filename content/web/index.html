Connect via ADB

  1. Install ADB on your machine

    Download from Android Developer Studio or use your package manager.

    Shell
    # Debian / Ubuntu
sudo apt install adb

# macOS
brew install android-platform-tools

# Windows — download SDK Platform Tools from:
# https://developer.android.com/studio#downloads

  2. Find the connection address

    In the LazyCat app detail page, locate the TCP ingress host and port for port 5555.

  3. Connect

    Shell
    adb connect <host>:<port>

# Verify connection
adb devices
# Expected output:
# <host>:<port>    device
If the connection fails, verify the ADB port is reachable from your network. Check firewall rules and ensure you are on the same local network or VPN.

Screen Mirroring with scrcpy

  1. Install scrcpy

    Shell
    # Debian / Ubuntu
sudo apt install scrcpy

# macOS
brew install scrcpy

# Windows
# Download from https://github.com/Genymobile/scrcpy/releases

  2. Launch scrcpy

    Shell
    scrcpy -s <host>:<port>

# Useful options:
scrcpy -s <host>:<port> --max-size 1024     # Limit resolution
scrcpy -s <host>:<port> --bit-rate 2M        # Limit bitrate
scrcpy -s <host>:<port> --no-audio            # Disable audio forwarding
scrcpy -s <host>:<port> --stay-awake          # Keep screen on
scrcpy -s <host>:<port> --video-codec h265    # H.265 hardware encoding
For smoother performance on slower networks, use --max-size 1024 --bit-rate 2M to reduce bandwidth. scrcpy defaults to H.264; use --video-codec h265 for better quality at lower bitrates.

Troubleshooting

ADB connection refused

  • Verify the redroid container is running
  • Confirm the host address and port are correct
  • Check that no firewall is blocking the TCP ingress port
  • Try adb kill-server then reconnect

scrcpy shows a black screen

  • Wait 1–2 minutes for the Android system to fully boot
  • Try adb reconnect to refresh the connection
  • Verify the container has sufficient memory allocated

Container fails to start

  • Host must have binder_linux and ashmem_linux kernel modules loaded
  • Verify: lsmod | grep -E "binder|ashmem"
  • Host must support privileged containers
  • Check container logs for detailed error messages

Host Requirements

Kernel Module Note The LazyCat host must have the required kernel modules. If the container fails to start, the host administrator may need to run:
Shell (host)
apt install linux-modules-extra-$(uname -r)
modprobe binder_linux devices="binder,hwbinder,vndbinder"
modprobe ashmem_linux
